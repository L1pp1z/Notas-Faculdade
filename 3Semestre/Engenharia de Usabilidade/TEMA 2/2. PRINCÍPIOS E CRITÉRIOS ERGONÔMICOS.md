Princípios e critérios ergonômicos são elementos que desenvolvedores devem seguir para garantir uma experiência mais agradável e interessante para o usuário no seu contato com a interface de qualquer sistema.
# CRITÉRIOS
## CONDUÇÃO
Seu objetivo é guiar os usuários ainda sem experiência no uso do sistema, a fim de que realizem suas tarefas.
- Fornecer um título para cada campo de entrada de dados.
- Dar um título a cada nova janela que se abra na aplicação. 
- Criar mecanismo de ajuda on-line para cada campo de informação e para a interface como um todo.
- Indicar formato nas entradas de dados. 
- Definir um agrupamento adequado dos itens na interface.
### SUBCRITÉRIOS
- <span style="color:rgb(255, 192, 0)">CONVITE:</span> Uma interface deve ser convidativa e apresentar informações claras.
- <span style="color:rgb(255, 192, 0)">AGRUPAMENTO E DISTINÇÃO POR LOCAL E FORMATO:</span> Uma interface deve orientar o usuário por meio da boa organização espacial e gráfica das informações.
- <span style="color:rgb(255, 192, 0)">LEGIBILIDADE:</span> Uma interface deve utilizar recursos que ajudem na leitura da tela.
- <span style="color:rgb(255, 192, 0)">FEEDBACK IMEDIATO:</span> Uma interface deve implementar mecanismos de respostas rápidas ao usuário em forma de diálogo.
## CARGA DE TRABALHO
Seu propósito é diminuir ao máximo a carga cognitiva que o usuário precisa para realizar seu trabalho. Para a otimização da carga de trabalho, alguns itens devem ser observados:
- <span style="color:rgb(255, 192, 0)">BREVIDADE:</span> Reduzir ao máximo o tempo de leitura dos itens expostos e digitação de informação na interface. Deve-se sempre colocar o mínimo de informação possível na tela.
- <span style="color:rgb(255, 192, 0)">CONCISÃO:</span> Diminuir o trabalho em relação às entradas e saídas do sistema.
- <span style="color:rgb(255, 192, 0)">AÇÕES MÍNIMAS:</span> Simplificar os passos para execução de uma tarefa.
## CONTROLE EXPLÍCITO
Seu objetivo é permitir que o usuário tenha controle sobre a interface do sistema.

## ADAPTABILIDADE
Sua meta é garantir a capacidade de o sistema se adaptar ao contexto e às preferências do usuário.
--- start-multi-column: 
```column-settings  
number of columns: 2

```

<span style="color:rgb(255, 192, 0)">Flexibilidade</span>
Tarefas podem ser realizadas de diversas formas, pois o usuário pode decidir como melhor executar determinada tarefa.

--- end-column ---

<span style="color:rgb(255, 192, 0)">UX – User Experience</span>
O sistema deve prever o atendimento a diferentes necessidades de distintos usuários. A linguagem utilizada para diferentes públicos-alvo e os tipos de informação apresentados para cada tipo de usuário podem ser previstos.

--- end-multi-column
## GESTÃO DE ERROS
Seu propósito é prevenir ou reduzir a ocorrência de erros, entendendo como e quando ocorrem.
- PROTEÇÃO CONTRA ERROS: O sistema deve prover mecanismos para detectar e prevenir erros.
- QUALIDADE DAS MENSAGENS DE ERROS: As mensagens de erros devem ser claras e legíveis, o que deve ser feito para evitar o erro e como corrigi-lo.
- CORREÇÃO DE ERROS: As ações que o usuário pode realizar para solucionar os erros.
## CONSISTÊNCIA/HOMOGENEIDADE
Seu objetivo é padronizar as interfaces de modo que os elementos sejam facilmente reconhecidos, localizados e lembrados.
## SIGNIFICADOS DE CÓDIGOS
Sua meta é garantir que o vocabulário utilizado em rótulos, títulos, cabeçalhos, mensagens e opções de menus esteja compatível com as figuras e os ícones utilizados.
## COMPATIBILIDADE
Sua finalidade é fazer com que a organização das saídas e entradas de uma aplicação se adequem às características do usuário, como hábitos, idade, competências, expectativas etc.
# USABILIDADE: CRITÉRIOS E REQUISITOS
Os critérios de usabilidade garantem que a interface seja eficiente, fácil de memorizar e ofereça uma experiência satisfatória às necessidades dos usuários.
## CRITÉRIOS
Alguns aspectos de usabilidade também devem ser observados na construção de uma interface de fácil uso.
- <span style="color:rgb(255, 255, 0)">FACILIDADE DE USO:</span> Permitir que o usuário realize suas tarefas já no primeiro contato com a interface do sistema.
- <span style="color:rgb(255, 255, 0)">EFICIÊNCIA:</span> Possibilitar que o usuário realize suas tarefas com o melhor rendimento e com o mínimo de erros possíveis.
- <span style="color:rgb(255, 255, 0)">MEMORIZAÇÃO:</span> Viabilizar que, mesmo depois de um período sem usar o sistema, o usuário consiga utilizá-lo com a mesma facilidade e rapidez.
- <span style="color:rgb(255, 255, 0)">TAXA DE ERROS:</span> Permitir que os próprios usuários resolvam, de maneira rápida e fácil, os erros. Verificar a quantidade de erros que os usuários estão cometendo e se eles são graves ou não.
- <span style="color:rgb(255, 255, 0)">SATISFAÇÃO:</span> Ser agradável e atender às necessidades de interação com o usuário.
## REQUISITOS
Os sistemas devem ser projetados visando atender as necessidades e expectativas dos usuários. Os requisitos desejáveis podem ser agrupados em duas categorias.

<span style="color:rgb(255, 192, 0)">Exibição da informação</span>
Consistência, feedback, níveis de habilidade e conhecimento humano, percepção humana, metáforas, minimização de carga de memória, classificação funcional dos comandos e projeto independente da resposta do monitor.

--- end-column ---

<span style="color:rgb(255, 192, 0)">Entrada de dados</span>
Mecanismos de ajuda, prevenção e tratamento de erros.

# TIPOS DE PROBLEMAS NO USO DE INTERFACE
Três tipos de problemas que podem ser encontrados pelos usuários durante o uso do software.
--- start-multi-column:  
```column-settings  
number of columns: 3  
largest column: null
```

<span style="color:rgb(255, 192, 0)">BARREIRAS:</span>
Os usuários não conseguem superar sem ajuda externa. Isso impossibilita a execução do trabalho.

--- end-column ---

<span style="color:rgb(255, 192, 0)">OBSTÁCULOS:</span>
Os usuários conseguem resolver após algumas tentativas e sem ajuda de terceiros. Isso atrasa a execução do trabalho do usuário.

--- end-column ---

<span style="color:rgb(255, 192, 0)">RUÍDOS:</span>
Problemas mais simples, pois não atrapalham nem atrasam a execução do trabalho do usuário, mas ele pode ficar com uma má impressão do sistema.

--- end-multi-column
# AVALIAÇÃO DA APLICAÇÃO DOS CRITÉRIOS ERGONÔMICOS EM UMA INTERFACE
A verificação e validação ergonômica ajudam a identificar e solucionar problemas de IHC antes que o usuário final os encontre. Há duas formas de avaliação: **heurística** ou por **inspeção**.
--- start-multi-column: 
```column-settings  
number of columns: 2  
largest column: null
```

<span style="color:rgb(255, 192, 0)">Avaliação heurística</span>
Produz um julgamento sobre as qualidades ergonômicas das interfaces. Deve ser feita por um grupo de especialistas, pois, por ser uma avaliação subjetiva, existem diferenças entre as opiniões de especialistas.

--- end-column ---

<span style="color:rgb(255, 192, 0)">Inspeções ergonômicas</span>
São vistorias baseadas em listas de verificação (checklists) com diferentes níveis de detalhe. Elas sistematizam o diagnóstico.

--- end-multi-column
