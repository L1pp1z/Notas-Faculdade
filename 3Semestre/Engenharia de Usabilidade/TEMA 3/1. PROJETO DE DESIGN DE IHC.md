# CARACTERÍSTICAS DO PROJETO
O projeto de design da interface humano-computador (IHC) visa elaborar um modelo conceitual de entidades e atributos do domínio e do sistema, estruturar as tarefas e projetar as interações e a interface de um sistema interativo que apoie os objetivos do usuário.

O projeto de design de interface se preocupa com muitos elementos, entre eles:
- Parte visual
- Usabilidade
- Arquitetura da informação
- Navegação
- Transição de telas
> [!tip] Dica
> Tudo deve ser pensado e planejado para que a experiência dos usuários seja de satisfação durante o uso do sistema. Vem daí a ideia de experiência do usuário ou UX.

## AFFORDANCE
O affordance é um termo que significa o conjunto de características do software perceptíveis pelo usuário e como realizá-las.
>[!tip] Dica
>Affordance é a capacidade que um objeto tem de ser reconhecido e utilizado exatamente para o que foi projetado, mas sem a necessidade de uma explicação prévia.

## COMUNICABILIDADE
A comunicabilidade é a propriedade de uma interface que permite transmitir, as finalidades e a lógica do design do sistema ao usuário. 
>[!QUOTE] (PRATES; BARBOSA, 2007)
> Se um usuário é capaz de compreender a lógica utilizada na concepção do sistema interativo, terá maiores chances de fazer um uso criativo, eficiente e produtivo dele.

## FASES DE UM PROJETO IHC
As fases de um projeto de IHC são fundamentais para garantir que o sistema atenda às necessidades reais dos usuários e suporte todos os requisitos de negócio.
- IDENTIFICAR AS NECESSIDADES E DEFINIR OS REQUISITOS: Entender os processos de trabalho dos usuário, verificar as necessidades reais
- DESENVOLVER DESIGNS ALTERNATIVOS: Explorar vários aspectos com relação ao visual e à usabilidade do software. Cenários são criados para avaliação e escolha dos usuários
- CONSTRUIR VERSÕES INTERATIVAS DOS DESIGNS: Desenvolvimento de protótipos para interação do usuário
- AVALIAR O DESIGN: Avaliação e ranqueamento do design dos protótipos
## DEFININDO AS REPRESENTAÇÕES DO PROJETO
Para registro das informações coletadas no processo de levantamento de requisitos precisamos de representações. Exemplos de representações e modelos:
- Perfil do Usuário
- Personas
- Cenários
- Análise de Tarefas
### PERFIL DO USUÁRIO
Consiste em uma representação detalhada das características e necessidades de um usuário típico de um sistema ou produto, visando conhecer o grau de domínio desse usuário em relação à área de atuação do sistema e em relação às tarefas que ele deverá realizar ao utilizar o sistema.

### PERSONAS
Personas são definidas, principalmente, por seus objetivos de atividade no sistema em geral. Cada projeto deve possuir seu próprio **elenco de personas**.

### CENÁRIOS
Os cenários podem ser utilizados para descrever como algo acontece, com bastante riqueza de detalhes, de modo que promova o melhor entendimento sobre a situação. Entretanto, o mais importante é que os cenários descrevam o comportamento dos atores (personas).
![[Pasted image 20260417202447.png]]
### ANÁLISE DE TAREFAS
Esta análise é utilizada para se ter um entendimento melhor sobre qual é o trabalho dos usuários. Nesse tipo de análise, o trabalho não é apenas elencar tarefas, mas entender como o sistema irá afetar esse trabalho realizado.
