# Protótipo
<span style="color:rgb(255, 192, 0)">CARACTERÍSTICAS:</span>
- É muito útil para a compreensão do sistema.  
- Serve de demonstração para os clientes.  
- É mais flexível para mudanças.  
- Quando está mais evoluído, pode ser integrado ao produto completo para uma entrega mais rápida da versão final.
- Apresenta uma abordagem útil para criar aplicações de comércio eletrônico e aplicativos de dispositivos móveis.

<span style="color:rgb(255, 192, 0)">VANTAGENS:</span>
- Melhor compreensão do sistema
- Facilidade de demonstração para clientes
- Flexibilidade para mudanças
- Integração gradual ao produto final aumentando a eficiência de desenvolvimento e agilidade na entrega

>[!info]
> Um dos aspectos mais importantes do modelo iterativo e incremental é garantir que os protótipos desenvolvidos sejam reutilizáveis para o projeto do sistema. 

>[!warning]
>A Rad foca o desenvolvimento rápido por meio de iterações frequentes e feedbacks contínuo.

# Rad
- Aplicação da metologia Joint Application Development (JAD)
- Conhecida também como Rapid Application Building (RAB)
- [[1. INTRODUÇÃO#-> Princípios|PRINCÍPIOS:]]
	- Estabelece dinâmica de grupo
	- Recursos audiovisuais
	- Processo Organizado e Racional
	- Escolha do Local
	- Documentação WYSIWIG (“O que você vê é o que você obtém”)
	- Desenvolvimento de Protótipos para chegar ao sistema final

<span style="color:rgb(255, 192, 80)">POSSUI 4 ELEMENTOS FUNDAMENTAIS: </span>
- Uso de Ferramentas para dar suporte ao desenvolvimento: 
	- Ferramentas CASE
- Metodologia bem definida:
	- Processo formal de desenvolvimentos com atividades e entregas intermediárias
- Pessoas:
	- Treinamento das pessoas na metodologia e nas ferramentas
- Gestão:
	- Gerenciamento feito com rapidez, através de oficinas de planejamento de requisitos e design de sistemas para extrair requisitos dos usuários

## Princípios, Ferramentas e Técnicas
Os Princípios facilitam a adaptação rápida às mudanças de requisitos e promovem colaboração entre devs e usuários.

### Princípios
Segundo Fitzgerald(1998), a RAD é baseada nos seguintes princípios básicos.

- <span style="color:rgb(255, 192, 0)">ENVOLVIMENTO ATIVO DOS USUÁRIOS:</span>
	- É necessário para reduzir problemas caros de obtenção de requisitos
	- Podem rejeitar os sistemas se não estiverem envolvidos no desenvolvimento
	- Oficinas de design de aplicativos conjuntos(JAD) e planejamento de requisitos
- <span style="color:rgb(255, 192, 0)">EQUIPES PEQUENAS COM PODER DE DECISÃO:</span> 
	- Redução de Ruídos de comunicação
	- Minimização de atrasos
	- Empoderar equipes aumentam as chances de cumprir os prazos
	- Podem tomar decisões sobre o design
- <span style="color:rgb(255, 192, 0)">ENTREGA FREQUENTE DE PRODUTOS:</span>
	- Reduzir o tempo de desenvolvimento
	- Prazos curtos
	- Entrega de produtos que satisfaçam os requisitos funcionais
- <span style="color:rgb(255, 192, 0)">DESENVOLVIMENTO INCREMENTAL E ITERATIVO:</span>
	- Sistemas evoluem de forma incremental, através da prototipagem iterativa
	- Cada nova iteração surgem novos requisitos que são incorporados ao sistema
	- Especificação de requisitos é um processo não determinístico e que evolui à medida que devs e users interagem com o protótipo do sistema
	- <span style="color:rgb(255, 192, 0)">ABORDAGEM TOP-DOWN:</span> 
		- Requisitos não precisam ser definidos no início, são especificados em um nível apropriado ao conhecimento disponível no momento
		- Sistemas são elaborados e feitos à medida que o conhecimento cresce
		- Todas as decisões são consideradas reversíveis
	- <span style="color:rgb(255, 192, 0)">FERRAMENTAS DE AUTOMAÇÃO(CASE):</span> 
		- Programas que facilitam a automação de processos e tarefas que ajudam na entrega dentro dos prazos
		- Facilitam a reutilização de componentes ao longo do projeto

### Ferramentas e Técnicas
Auxilia no desenvolvimento de aplicações rapidamente como:
- Integração de dados
- Ambientes de desenvolvimento
- Coleta de requisitos
- Modelagem de dados
- Geração de código

A ideia do uso das técnicas de RAD é de otimizar os resultados obtidos dentro do tempo estimado, permite o usuário ter uma visão clara de como o sistema está progredindo

##### <span style="color:rgb(255, 192, 0)">TÉCNICAS PARA UTILIZAÇÃO:</span>
- <span style="color:rgb(255, 192, 0)">MODELO CBD</span>
	- Descreve componentes antigos que podem ser reutilizados como novos
- <span style="color:rgb(255, 192, 0)">REPOGUARD</span>
	- Framework para integração de ferramentas de desenvolvimento com repositores de código-fonte
- <span style="color:rgb(255, 192, 0)">ADIÇÃO DINÂMICA ÁGIL</span>
	- Integração do ágil para tornar o projeto adaptável
- <span style="color:rgb(255, 192, 0)">CAMADAS PARA DESENVOLVIMENTO RÁPIDO</span>
	- Baseado em camadas e segue o XP (Extreme Programming
	- Visa Maximizar a qualidade
	- Responder rapidamente às mudanças nos requisitos do cliente
- <span style="color:rgb(255, 192, 0)">ANÁLISE DE PROJETO BASEADO EM SIMULAÇÃO</span>
	- Desenvolvimento de ferramentas ágeis baseadas em simulação
- <span style="color:rgb(255, 192, 0)">USO DE AJAX</span> 
	- Prototipagem rápida em aplicativos e ferramentas da web
- <span style="color:rgb(255, 192, 0)">DESENVOLVIMENTO DE APPS MULTIUSERS EM AMBIENTE DISTRIBUÍDO RAPIDAMENTE</span>
	- Middleware da comunicação
- <span style="color:rgb(255, 192, 0)">PROGRAMAÇÃO EXTREMA</span>
	- Refere-se à adição de reutilização ao XP
