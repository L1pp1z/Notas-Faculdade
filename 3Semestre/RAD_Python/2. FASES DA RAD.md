## Fases James Martin (1991)
Na abordagem de James Martin, a metologia RAD possui 4 etapas
- Planejamento de Requisitos
- Design do Usuário
- Construção
- Transição

##### <span style="color:rgb(255, 192, 0)">PLANEJAMENTO DE REQUISITOS</span>
- Focado em determinar metas, expectativas e potenciais problemas
- Usuários, gerentes e devs estudam sobre:
	- Necessidades de negócios
	- Escopo do projeto
	- Restrições e requisitos do sistema
##### <span style="color:rgb(255, 192, 0)">DESIGN DO USUÁRIO</span>
- São desenvolvidos modelos e protótipos, através da interação de users e devs. Combinam técnicas JAD e ferramentas CASE
##### <span style="color:rgb(255, 192, 0)">CONSTRUÇÃO</span>
- Protótipos são desenvolvidos. Interação para sugestões sobre alterações, ajustes ou melhores à medida que o projeto é desenvolvido
- Converte o protótipo aprovado em um modelo de trabalho final
##### <span style="color:rgb(255, 192, 0)">TRANSIÇÃO</span>
- Realiza o processamento de dados, coleta e organiza informações importantes
- Execução de testes para garantir funcionalidades e segurança
- Treinamento do usuário
- Lançamento do Produto

## Fases de James Kerr (1994)
Na abordagem de James Kerr, a metologia RAD possui 5 etapas
- Modelagem de Negócios
- Modelagem de Dados
- Modelagem de Processos
- Geração da Aplicação
- Teste e Modificação

##### <span style="color:rgb(255, 192, 0)">MODELAGEM DE NEGÓCIOS</span>
- Informações sobre os requisitos funcionais do sistema são coletadas
- O Modelo de negócio do produto é definido em termos de fluxo de informações
- Informações são combinadas para criar a documentação que modelará o ciclo de vida dos dados
- Análise comercial para identificar dados essenciais para os negócios
##### <span style="color:rgb(255, 192, 0)">MODELAGEM DE DADOS</span>
- Todas as informações são analisadas e formam objetos de dados essenciais
- Informações são agrupadas de maneira útil
- Examinam a qualidade de cada grupo de informação
- Faz-se um mapeamento que relaciona os grupos e definem relacionamentos
- Identifica e define atributos de todos os conjuntos de dados
- Estabelecer e detalhar a relação entre esses objetos
##### <span style="color:rgb(255, 192, 0)">MODELAGEM DE PROCESSOS</span>
- Todos os grupos de dados são analisados e convertidos em informações úteis
- Pode-se realizar mudanças e otimizações
- Descrições dos grupos são criadas
##### <span style="color:rgb(255, 192, 0)">GERAÇÃO DA APLICAÇÃO</span>
- Todas as infos coletadas são codificadas
- Construção do sistema para criar o protótipo
- Modelos de dados são transformados em protótipos
- Sistema real é construído
- Codificação é feita usando ferramentas de automação
##### <span style="color:rgb(255, 192, 0)">TESTE E MODIFICAÇÃO</span>
- Testes nos protótipos criados
- Cada módulo é testado
- Protótipos são testados independentemente durante cada iteração
- Fluxo de dados e interfaces são exaustivamente testados
- Risco de problemas importantes é minimizado

>[!tip]
>Um dos maiores benefícios da RAD são os comentários dos usuários através da constante interação.

## Ciclo de Desenvolvimento
É iniciado com um planejamento conciso em que as necessidades e os requisitos são definidos em colaboração com usuários finais como:
- Metas
- Expectativas
- Cronogramas
- Orçamento
O cliente deve fornecer a visão do produto e, junto com outras partes interessadas, realizar pesquisas para finalizar os requisitos, garantindo a aprovação de todos.

##### <span style="color:rgb(255, 192, 0)">LEVANTAMENTO DE REQUISITOS</span>
- Pesquisa do ambiente interno
- Desenvolvimento do escopo do sistema
- Processos do negócio e dados são utilizados
- Características como benefícios, custos e riscos são identificadas
- Formalização do escopo
- Estimatima-se custo e duração do projeto
##### <span style="color:rgb(255, 192, 0)">OFICINAS JAD</span>
- Revisões e detalhamento do escopo do sistema
- Elementos UI/UX, layouts provisórios e relatórios são desenvolvidos
- Conclusão do design
- Mapeamento das funcionalidades do sistema
##### <span style="color:rgb(255, 192, 0)">VALIDAÇÃO DOS PROTÓTIPOS</span>
- Consistência é validada através de sucessivas iterações
- Identificação e correção de problemas e ajustes
- Resultados das iterações são incorporados ao projeto e ao plano de transição para construção
##### <span style="color:rgb(255, 192, 0)">BANCO DE DADOS</span>
- Construção do banco de dados
- Testes são executados com dados que simulem situações reais, para detectar falhas
##### <span style="color:rgb(255, 192, 0)">TRANSIÇÃO</span>
- Treinamento dos usuários antes que seja colocado em produção
- Sistema em produção e após finalização, instala-se no cliente
- Configuração dos requisitos
- Aceitação do sistema por parte do usuário
